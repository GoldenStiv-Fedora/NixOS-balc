# 🚀 NixOS-balc: GitOps Управление Парком ПК

Этот репозиторий содержит централизованную конфигурацию для развертывания и управления парком из **50+ рабочих станций** на базе **NixOS**. Система спроектирована по принципу "Infrastructure as Code" (IaC) с использованием механизма **GitOps**: любые изменения в репозитории автоматически применяются на всех компьютерах сети.

## 🛠 Ключевые возможности

*   **Автоматическая синхронизация**: Фоновое обновление конфигурации из GitHub каждые 3 часа.
*   **Поддержка Rutoken S (HID) и Lite (CCID)**: Глубоко пропатченный драйвер для работы со старыми HID-токенами, включая автоматический проброс в RDP.
*   **Безопасный VPN & RDP**: Скрипт-обертка для быстрого подключения к удаленным рабочим столам через L2TP/IPSec VPN с использованием графического интерфейса.
*   **Автоматизация XFCE**: Динамическая смена обоев из Git-репозитория и автоматическое создание ярлыков RDP для всех пользователей.
*   **Самообслуживание**: Еженедельная очистка системы от мусора и оптимизация хранилища Nix Store.

## ⚙️ Описание модулей

### 1. `git-sync.nix` (Сердце системы)
Реализует механизм **"Pull-модели"**. Создает системный таймер, который каждые 3 часа клонирует репозиторий в `/etc/nixos/.sync`, проверяет наличие обновлений и выполняет `nixos-rebuild switch`.
*   **Защита от дурака**: Использует `rsync` с исключением `hardware-configuration.nix`, что гарантирует сохранность настроек загрузчика на разном "железе".

### 2. `rutoken.nix` (Работа с ЭЦП)
Уникальный модуль для NixOS, решающий проблему старых драйверов Rutoken S.
*   Использует `autoPatchelfHook` для исправления зависимостей проприетарной библиотеки `librutokens.so`.
*   Добавляет поддержку `libusb-compat` для работы HID-устройств.
*   Настраивает `pcscd` и правила `udev` для бесшовной работы токенов.

### 3. `vpn.nix` (Удаленный доступ)
Содержит скрипт `connect-rdp`, который через диалоговые окна `zenity` запрашивает учетные данные, поднимает VPN-туннель и запускает FreeRDP.
*   **Важный фикс**: Принудительно загружает библиотеку PCSC через `LD_PRELOAD` для корректного проброса токенов в сессию Windows.

### 4. `users.nix`
Управление пользователями (`Balc`, `User`) и автоматизация интерфейса. Скрипт `setup-xfce-wallpaper` при каждом входе выбирает случайное изображение из папки `imag/` и устанавливает его на рабочий стол.

## 🤖 Автоматизация GitHub Actions (`update.yml`)

В репозитории настроен Workflow, который работает как **виртуальный системный администратор**:
*   **Частота**: Запускается в **02:00 каждую ночь**.
*   **Что делает**: Обновляет `flake.lock`, проверяя наличие новых версий системных пакетов.
*   **Механизм заглушек**: При обновлении Workflow создает временные пустые файлы (`users.nix`, `rutoken.nix` и т.д.), чтобы `nix flake update` не падал из-за отсутствия импортируемых модулей, которых нет в корне репозитория.
*   **Результат**: Каждое утро ваш парк ПК готов получить обновления безопасности без ручного вмешательства.

## 🚀 Быстрое развертывание (Deployment)

Для ввода нового ПК в эксплуатацию достаточно установить чистую NixOS и выполнить одну "Золотую команду":

```bash
sudo nix-env -iA nixos.git nixos.rsync && \
sudo mkdir -p /etc/nixos/.sync && \
sudo git clone --filter=blob:none --sparse https://github.com/GoldenStiv-Fedora/NixOS-balc.git /etc/nixos/.sync && \
cd /etc/nixos/.sync && \
sudo git sparse-checkout set configs imag && \
sudo rsync -a /etc/nixos/.sync/configs/ /etc/nixos/ --exclude=hardware-configuration.nix && \
sudo nixos-rebuild switch --flake /etc/nixos#nixos --option experimental-features "nix-command flakes"

```

### Что произойдет после запуска:
1.  Система установит инструменты клонирования.
2.  Безопасно скопирует конфигурации (не затронув UUID ваших дисков).
3.  Развернет графическую оболочку, драйверы, VPN и скрипты.
4.  Через 10 минут после старта докачает библиотеку обоев из папки `imag/`.
